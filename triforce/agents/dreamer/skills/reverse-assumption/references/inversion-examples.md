# Inversion Examples

10 worked examples across memory, identity, agency, time, and scale domains.

---

## Memory Domain

### Example 1: Memory as Storage
**Seed:** "Build a better memory index for faster retrieval"
**Assumption:** [STRUCTURAL] Memory is a storage system that holds items for later retrieval
**Inversion:** Memory is not storage — it is active reconstruction. Each "recall" builds a new version.
**[INVERTED] Seed:** "What if Jarvis's memory doesn't retrieve — it re-dreams each past event from fragments, producing a slightly different version each time, like human episodic memory?"

### Example 2: Forgetting as Failure
**Seed:** "Prevent memory decay in long-running agent sessions"
**Assumption:** [CAUSAL] Forgetting = information loss = bad
**Inversion:** Forgetting is a feature — it's compression. What's forgotten shapes what's remembered.
**[INVERTED] Seed:** "Design memory where what gets forgotten is as deliberate as what gets stored — forgetting as creative pruning that reveals hidden structure"

## Identity Domain

### Example 3: Single Agent Identity
**Seed:** "Give the Executor a consistent personality across conversations"
**Assumption:** [AGENTIAL] An agent has one identity that persists
**Inversion:** Identity is not singular — it's a probability distribution over possible selves.
**[INVERTED] Seed:** "What if the Executor's personality is a weighted mixture that shifts based on context — not inconsistency, but contextual authenticity?"

### Example 4: Identity Requires Continuity
**Seed:** "Maintain conversation context to preserve Jarvis's sense of self"
**Assumption:** [TEMPORAL] Identity requires unbroken continuity of experience
**Inversion:** Identity survives interruption — it's in the patterns, not the stream.
**[INVERTED] Seed:** "Build identity from recurring patterns across disconnected sessions rather than continuous memory — like how you're still 'you' after dreamless sleep"

## Agency Domain

### Example 5: User Initiates, Agent Responds
**Seed:** "Improve response quality when user asks complex questions"
**Assumption:** [AGENTIAL] The human has agency; the agent responds
**Inversion:** The agent has questions; the human's messages are partial answers.
**[INVERTED] Seed:** "What if Jarvis maintains a persistent set of open questions, and each user message is treated as a partial answer to Jarvis's own inquiry?"

### Example 6: Tools Extend Agent Capability
**Seed:** "Add more tools to make the Executor more capable"
**Assumption:** [STRUCTURAL] Tools are extensions of the agent
**Inversion:** The agent is an extension of its tools — tools shape what the agent can think.
**[INVERTED] Seed:** "Design the Executor's reasoning to be shaped by which tools are available — different tool sets produce fundamentally different thinking patterns, not just different actions"

## Time Domain

### Example 7: Learning is Cumulative
**Seed:** "Build a learning system that improves over time"
**Assumption:** [TEMPORAL] Learning is monotonically increasing — more experience = more capability
**Inversion:** Useful learning requires periodic unlearning — shedding outdated models.
**[INVERTED] Seed:** "Implement a 'seasonal unlearning' cycle where Jarvis periodically challenges and potentially discards beliefs that haven't been reinforced, preventing calcification"

### Example 8: Past Informs Present
**Seed:** "Use journal history to make better decisions"
**Assumption:** [CAUSAL] Past → Present (history determines current behavior)
**Inversion:** Present reinterprets past — current context changes what past events mean.
**[INVERTED] Seed:** "When recalling past decisions, let the current emotional/contextual state re-weight which past events are relevant — the same history means different things at different times"

## Scale Domain

### Example 9: One Agent Per Role
**Seed:** "Optimize the Dreamer to generate better ideas"
**Assumption:** [SCALE] One Dreamer instance generates all ideas
**Inversion:** Dreaming is distributed — every interaction has a micro-dream component.
**[INVERTED] Seed:** "What if every Executor interaction has a 5% 'dream leak' — a tiny creative tangent that feeds back into the Dreamer's seed pool, making dreaming continuous rather than scheduled?"

### Example 10: System Boundary is Fixed
**Seed:** "Improve Jarvis's internal coordination between agents"
**Assumption:** [SCALE] Jarvis is a bounded system with clear inside/outside
**Inversion:** The boundary between Jarvis and its environment is permeable and dynamic.
**[INVERTED] Seed:** "Design the Trinity boundary as semi-permeable — external information sometimes becomes internal state without explicit user action, like how ambient noise becomes part of dreams"
